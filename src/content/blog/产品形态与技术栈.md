## 一、产品组成

常见的软件开发产品为以下四层的组合体

```text
用户终端
Web / 手机 App / 桌面端 / 小程序 / CLI / 硬件设备
        ↓
业务服务
用户、权限、订单、支付、内容、搜索、消息、文件、推荐
        ↓
数据系统
数据库、缓存、消息队列、搜索引擎、对象存储、数据仓库
        ↓
基础设施
Linux、Docker、云服务器、CI/CD、监控、Kubernetes
```

| 产品形态      | 常见产品                      | 主要语言                           | 常见技术栈                                          |

| Web 应用    | 电商、在线教育、社交网站、在线文档、SaaS    | TypeScript 为主                  | React/Vue/Angular + Java/Go/Node.js/Python 后端  |
| 企业管理系统    | CRM、ERP、OA、财务、供应链、内部后台    | Java、C#、TypeScript             | Spring Boot、ASP.NET Core、React/Vue、MySQL、Redis |

| 桌面软件      | VS Code、聊天工具、笔记软件、设计工具    | TypeScript、C#、C++、Rust、Swift   | Electron、Tauri、.NET、Qt、SwiftUI                 |

| AI 产品     | AI 聊天、知识库、智能搜索、图像生成、Agent | Python、TypeScript              | PyTorch、FastAPI、LLM API、向量数据库、React/Next.js    |

这里的分类不是互斥的。一个成熟产品通常包含多种形态，例如一个电商平台可能同时拥有：

```text
React Web 商城
Kotlin / Swift 手机 App
Vue 管理后台
Java / Go 后端服务
Python 推荐系统
MySQL + Redis + Kafka
Docker + Kubernetes 部署
```

## 三、常见的几条技术路线

### 1. Web 前端路线

现代 Web 开发的核心语言是：

```text
HTML + CSS + JavaScript / TypeScript
```

常见组合：

```text
TypeScript
├── React
│   └── Next.js
├── Vue
│   └── Nuxt
├── Angular
└── Svelte
```

TypeScript 的重要性还在提升。GitHub 的 2025 Octoverse 数据显示，按 GitHub 贡献者数量计算，TypeScript 在 2025 年首次超过 Python 和 JavaScript，成为排名第一的语言；Python 和 JavaScript分别位列第二、第三。需要注意，这只是 GitHub 的统计方法，不代表所有语言排行榜都会得出相同结论。([The GitHub Blog][1])

2025 Stack Overflow 调查中，Node.js、React、Next.js、Express、ASP.NET Core、Angular、Vue、Spring Boot 和 FastAPI 都是较常见的 Web 技术，说明当前 Web 产品并不存在唯一技术栈，而是形成了多个成熟生态。([Stack Overflow Insights][2])

适合：

* 网站与管理后台
* SaaS 产品
* 独立开发
* 全栈开发
* 浏览器插件
* Electron 桌面软件

---

### 2. Java 企业后端路线

典型技术栈：

```text
Java
├── Spring Boot
├── Spring MVC
├── Spring Security
├── MyBatis / MyBatis-Plus / JPA
├── MySQL / PostgreSQL
├── Redis
├── RabbitMQ / Kafka
├── Maven / Gradle
└── Docker
```

典型产品：

* 电商订单系统
* 银行和金融系统
* 企业管理平台
* 支付系统
* 物流和供应链
* 大型互联网业务后台

Java 的特点不是“适合做页面”，而是适合承担复杂的业务规则、事务、权限、数据处理和长期维护。

因此你现在将 Java 作为第一门后端语言，是一个比较稳妥的选择。它能让你系统学习：

```text
HTTP → 数据库 → 缓存 → 权限 → 并发 → 消息队列 → 分布式系统
```

### 3. Go 云服务与基础设施路线

典型技术栈：

```text
Go
├── Gin / Echo / Fiber
├── GORM / sqlx
├── PostgreSQL / MySQL
├── Redis
├── gRPC
├── Kafka
├── Docker
└── Kubernetes
```

适合：

* API 服务
* 微服务
* 云原生平台
* 网络服务
* 即时通信
* 网关
* CLI 工具
* 基础设施软件

很多基础设施项目和开发者工具喜欢使用 Go，是因为它编译部署简单、并发模型直接、生成单个可执行文件比较方便。

不过 Go 通常更适合作为你的**第二门后端语言**。先通过 Java 建立完整的后端体系，再学习 Go，你会更容易理解 Go 为什么强调简洁、组合和显式错误处理。

### 4. Python AI、数据和快速后端路线

典型技术栈：

```text
Python
├── FastAPI / Django / Flask
├── PyTorch
├── Pandas / NumPy
├── Jupyter
├── PostgreSQL
├── Redis
├── Celery
└── LLM / RAG / Agent 框架
```

适合：

* 人工智能
* 机器学习
* 数据分析
* 自动化脚本
* 爬虫
* AI API
* 快速验证产品

GitHub 2025 数据中，Python 依然在 AI 和数据科学领域占据主导位置，Jupyter Notebook 也是 AI 项目常用的探索环境。([The GitHub Blog][1])

现在很多 AI 产品实际上是：

```text
TypeScript / React：用户界面
Python / FastAPI：AI 服务
Java / Go：核心业务系统
PostgreSQL：业务数据
向量数据库：知识检索
对象存储：文件与模型
```

也就是说，AI 产品并不等于“整个系统全部用 Python”。

### 6. 手机原生与跨平台路线

#### iOS 原生

```text
Swift + SwiftUI + Xcode
```

Apple 将 SwiftUI 定位为面向 Apple 各平台构建新应用的现代方案，可以覆盖 iOS、iPadOS、macOS、watchOS 和 visionOS。([Apple Developer][3])

#### Android 原生

```text
Kotlin + Jetpack Compose + Android Studio
```

Google 推荐现代 Android 项目使用 Kotlin，并为 Kotlin 和 Jetpack Compose 提供一等支持。([Android Developers][4])

#### 跨平台

```text
Flutter：Dart
React Native：TypeScript / JavaScript
Kotlin Multiplatform：Kotlin
```

Flutter 可以从一套 Dart 代码构建移动端、Web 和桌面应用；React Native 使用 React 编写界面并调用原生平台能力；Kotlin Multiplatform 则主要帮助 Android 和 iOS 共享业务逻辑，必要时也可以共享 UI。([flutter.dev][5])

---

### 7. 桌面客户端路线

桌面应用主要有三种思路。

**Web 技术包装：**

```text
TypeScript + React/Vue + Electron
```

Electron 把 Chromium 和 Node.js 放进桌面应用中，让一套 `HTML`、`CSS`、`JavaScript` 代码运行在 Windows、macOS 和 Linux。([Electron][6])

**Web 前端加轻量原生容器：**

```text
TypeScript + React/Vue + Tauri
Rust 负责底层能力
```

**完全原生：**

```text
Windows：C# + WPF / WinUI
macOS：Swift + SwiftUI / AppKit
跨平台：C++ + Qt
```

像==编辑器、聊天工具、笔记软件和数据库客户端==，经常会选择 Electron 或 Tauri；对系统集成、性能和原生体验要求很高时，则会采用 C++、C# 或 Swift。

## 四、必学的公共技术

无论你选 Java、Go 还是 Python，下面这些公共技术都会反复出现，几乎所有现代产品都会用到。

### 网络与接口

```text
HTTP / HTTPS
REST API
JSON
WebSocket
RPC / gRPC
身份认证：Session、JWT、OAuth 2.0
```

### 数据系统

```text
关系数据库：PostgreSQL、MySQL
缓存：Redis
文档数据库：MongoDB
搜索：Elasticsearch / OpenSearch
消息队列：Kafka、RabbitMQ
对象存储：S3、MinIO
```

2025 Stack Overflow 调查中，PostgreSQL 在数据库类别里同时拥有最高的“想使用”和“继续使用”比例；Redis、MySQL、MongoDB 和 SQLite 也依然是主流数据技术。([Stack Overflow Insights][2])

### 工程与部署

```text
Git / GitHub
Linux
Docker
CI/CD
Nginx
云服务器
日志与监控
自动化测试
```

Docker 已经属于非常通用的工程技能。2025 Stack Overflow 调查中，Docker 的使用比例达到 71.1%，Kubernetes 为 28.5%；这说明 Docker 更接近普遍基础能力，而 Kubernetes 更多用于复杂服务和较成熟的团队。([Stack Overflow Insights][2])

大型云原生组织对 Kubernetes 的采用率确实很高，但这不代表每个个人项目都应该使用 Kubernetes。CNCF 的 2025 年度调查显示，在其调查的容器用户中，82% 已经在生产环境运行 Kubernetes；这个数据更能反映企业和云原生团队，而不是初学者项目。([CNCF][7])

## 六、当前学习路线

你不需要同时学习所有产品形态。可以先建立一个主干，再了解其他分支。

```text
主干：Java 后端
│
├── Java 基础
├── MySQL + SQL
├── Spring Boot
├── HTTP + REST API
├── MyBatis / JPA
├── Redis
├── Git + Linux
├── Docker
└── 完整项目
```

同时补充必要的前端能力：

```text
HTML + CSS + JavaScript
        ↓
TypeScript
        ↓
Vue 或 React
```

之后再选择一个扩展方向：

```text
Java + Go
→ 微服务、云原生、基础设施

Java + TypeScript
→ Web 全栈、SaaS、独立开发

Java + Python
→ AI 应用、推荐系统、数据平台

Java + Kotlin
→ JVM 生态、Android、跨平台
```

对你来说，现阶段最值得建立的不是“会很多语言”，而是掌握一套完整的软件产品链路：

```text
做出页面
→ 调用接口
→ 编写后端业务
→ 读写数据库
→ 登录和权限
→ 缓存与消息
→ 测试
→ 部署上线
```

一旦完整走通这条链路，再学习 Go、Python 或 TypeScript，主要是在更换语言和框架，而不是从头重新学习软件开发。
